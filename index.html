준비 중
How It Works

작동 방식

음성 또는 UI로 목적지를 입력하면, 실외·실내 환경에 맞춰 자동으로 판단하며 주행합니다.

1

목적지 입력

사용자가 UI 터치 또는 음성 명령으로 목적지를 입력합니다. 음성 인식은 고정형 마이크와 클라우드 서버를 사용합니다.

UI / 음성 인식 (클라우드)
2

경로 생성

Tmap API로 실외 보행 경로를 요청하고 수신합니다.

Tmap API · GPS
3

자율 주행 시작

경로를 따라 주행하면서 카메라와 라이다가 실시간으로 장애물을 감지하고 우회합니다.

YOLO26n · Depth Anything V2 · 2D Lidar
4

실내 진입 시 자동 전환

실내 진입이 감지되면 라이다 우선·카메라(SLAM) 보조 모드로 전환해 실내 위치를 파악하고 경로를 추종합니다.

2D Lidar 우선 · SLAM 보조
5

목적지 도착

목적지에 도착하면 음성 및 화면으로 안내합니다.

TTS · 화면 알림
실외 주행 — 명령 우선순위
1순위카메라 데이터 (Depth · 장애물 감지 · 우회 판별)
2순위라이다 데이터 (실제 장애물 거리 측정)
3순위GPS — Tmap 경로안내 API
실내 주행 — 명령 우선순위
1순위라이다 데이터 (실내 장애물 거리)
2순위카메라 데이터 (SLAM 위치 보정)
3순위GPS 측위
Technology

기술별 주요 기능

각 센서와 소프트웨어가 분담하는 역할을 기술별로 정리했습니다.

📷

카메라 (Depth + SLAM)

  • YOLO26n 실시간 객체 감지 (장애물·사람·신호 등)
  • Depth Anything V2 — 단안 카메라 거리 추정
  • SLAM — 실내 지도 생성 및 자기 위치 추정
  • 실외 1순위 / 실내 2순위 센서
  • Jetson Orin Nano에서 TRT FP16 최적화
Python · OpenCV · PyTorch · Ultralytics
📡

2D 라이다

  • 실외 장애물 실측 거리 측정
  • 실내 장애물 회피 1순위 센서
  • 카메라 오감지 보완 역할
  • 저가 2D Lidar로 3D Lidar 대비 경제성 개선
Serial · 직접 드라이버
🗺️

GPS + Tmap API

  • 실외 보행 경로 생성 및 안내
  • 목적지까지 턴-바이-턴 경로 수신
  • 카메라/라이다 감지 없으면 GPS 경로 주행
  • 실외 3순위 명령 소스
Tmap API · HTTP REST
🎤

음성 인식

  • 고정형 마이크로 음성 입력 수집
  • 클라우드 서버에서 STT 처리 후 텍스트 반환
  • 음성으로 목적지 입력, 주행 명령 수행
Cloud STT · requests

Differentiation

기존 제품과의 차이점

기존 자율주행 휠체어와 세 가지 관점에서 비교합니다.

01 — 경제성

3D Lidar 없이도 동작

고가의 3D Lidar 대신 카메라(Depth·SLAM)와 저가 2D Lidar를 조합해 동등한 장애물 회피 성능을 달성했습니다.

02 — 완전 자율주행

목적지까지 개입 없이

기존 제품은 고정 장소 이동 또는 단순 장애물 회피에 그쳤지만, 본 기기는 실외·실내를 막론하고 목적지까지 완전 자율 주행합니다.

03 — 추가 장착형

기존 휠체어에 탈부착

새 휠체어를 구입할 필요 없이 기존 수동·전동 휠체어에 모듈을 장착하는 방식으로 도입 비용을 낮췄습니다.
